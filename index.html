Odoo setup and start on docker

Odoo's official documentation and getting started tutorial assumes that your environment is a standard Odoo laptop with Linux Mint installed

What if you are using a different environment like some other version of Linux, or maybe an entirely different system like Mac, or even Windows? It would be quite difficult to continue following the tutorials in the documentation!

One solution to this would be to use Docker.

Although the documentation provides a brief explanation on how to install Odoo in a docker container, it doesn't go further into how to practically add a new Odoo app in that environment.

In this tutorial we will go over on how to do that, after having also cleared out the Odoo installation process with docker.


Install Odoo with Docker


Install Docker

To install Docker in your environment, look at the documentation.

Install Odoo


An official way to install Odoo on docker is available here but we will simplify it a little bit, just for the matter of getting started, then show how to start a new Odoo application.

You would still be required to have some basic knowledge on how docker and docker-compose work!


edit docker-compose.yml


Here we are creating a docker-compose.yml configuration file in order to setup all the services needed for our application to run:


version: '3.1'
services:
  web:
    image: odoo:latest
    depends_on:
      - db
    ports:
      - "8069:8069"
    volumes:
      - odoo-web-data:/var/lib/odoo
      - ./config:/etc/odoo
      - ./addons:/mnt/extra-addons
    networks:
      - my_app_network
  db:
    image: postgres:13
    ports:
      - "6432:5432"
    environment:
      - POSTGRES_DB=postgres
      - POSTGRES_PASSWORD=odoo
      - POSTGRES_USER=odoo
      - PGDATA=/var/lib/postgresql/data/pgdata
    volumes:
      - odoo-db-data:/var/lib/postgresql/data/pgdata
    networks:
      - my_app_network
volumes:
  odoo-web-data:
  odoo-db-data:

networks:
  my_app_network:


In this configuration file we are defining several important entries like:

  • services which consist of
    • odoo itself, and
    • a postgres database that serves the data,
  • some volumes that help map data from our host machine to both ➝
    • the postgres database container and
    • the odoo docker container
  • a network that helps communicate between the two containers.

There is also one important entry that is very important to remember which is the ./addons:/mnt/extra-addons under services➝web➝volumes; this is what is going to allow us to add (and develop) new modules onto the Odoo framework.


run docker-compose

After creating the docker-compose.yml configuration file, we just run the follwing command in the same folder than the docker-compose.yml file.


docker-compose up

You can see logs of the command showing in the command prompt / terminal. To avoid showing those logs but run the app in the background, you can just add the -d flag to the above command


docker-compose up -d


This command will create as you notice several folders:

  • config, and
  • addons

to which we will come back later when creating a new app/module.


Login


Now let's go to localhost:8069 as mentioned in the above docker-compose.yml file.

This shows a login page:



This means Odoo is successfully running now on your computer. All you need to do is login.


Here the Master Password will be prefilled for you by default and you will indicate the Database Name as the same name you provided in the definition of the docker-compose.yml file (i.e. db in our case). Then chose a password that you remember, that will help you login the next time. Finally you can optionally fill in the rest of the information

A New Application


Now we are going to transition to starting our first odoo application in this docker setting.

Remember when we ran the previous command, it created two empty folders in our local directory. The first one addons is where we are going to define our custom odoo modules; the second one config is where we will configure the paths to our modules so that they can be installed inside our odoo framework (here the odoo docker container). This is possible because we declared those two folders as volumes in our docker-compose.yml file.


A new module


To create a new module in Odoo we need to go inside the addons directory and add a new directory which will correspond to our new module. Inside that new directory we will need to have two files at least: the __init__.py that indicates that this directory is just a python package like any other; and the __manifest__.py file where we add metadata relating to our module; this metadata, like the name of the module, is what shows up when the module is added to odoo framework.

So let's create a simple module named estate, to be consistent with the official odoo documentation.

It will look like this:

estate
	__init__.py
	__manifest__.py


The __init__.py file may be empty, but the __manifest__.py file usually looks like this but for simplicity we will add just one entry like the name of the module


__manifest__.py :

{
	"name": "Real Estate"
}

Update list of apps in odoo


After editing the configuration file, you might need to restart docker containers. Successively type the following commands:


docker-compose down


docker-compose up -d


Go tho localhost:8060 once again and login as usual.


Since we have added a new module we should be able to see it among the available apps. But before that we need to update the apps list first. To do so we need to enable to activate debug mode; click Ctr+k and start typing debug... an option shows up saying Activate debug mode , click on it then click on Update Apps List, then on Update . Now you can search for your new module in the search bar: type estate then enter, our new module should show up and ready to install just like any other odoo module or application.


We have successfully installed odoo on docker, started a new odoo application in that envrionment. The next steps would be to follow the documentation from there on, after this chapter.
